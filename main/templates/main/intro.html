사용 방법 안내


{{ url }}


이미지 가져오는 방법 : 서버 링크 + 이미지 경로
작동 방법 안에 쓰여져있는 한글은 변수를 입력하는 칸
리턴 정보중 소괄호 안에 있는 정보들은 여러개가 주어질 수 있음
id는 사용자에게 보여지면 안됨

게임 리스트 표시를 위한 게임 정보
작동 방법 : game_list
ex : {{url}}/game_list
리턴 정보 : (게임id, 게임 이름, 게임사진 작은버전)
게임은 br태그로 구분되어 있으며 게임 정보는 p태그로 구분되어서 제공됨. 인기 많은순으로 제공되며 최대 100개를 리턴함

게임 세부 정보
작동 방법 : game/게임id
ex : {{url}}/game/1
리턴 정보 : 게임id, 게임 이름, 게임 이용 가능 최소 인원수, 게임 이용 가능 최대 인원수, 게임 소개글, 게임사진 큰버전
해당 정보가 p태그로 구분되어서 제공됨

사용자와 가까운 보드게임 카페 안내
작동 방법 : vicinity_cafe?coords={user_latitude, user_longitude}
ex : {{url}}/vicinity_cafe?coords=127.57234,56.7923
리턴 정보 : (카페 id, 카페 이름, 카페 위도, 카페 경도)
해당 정보가 br태그와 p태그로 구분되어 있으며, 사용자에게 약 2km 내에 있는 카페를 가장 가까운 순으로 최대 5개를 반환함

위시리스트 정보
작동 방법 : get_wishlist?user_id={user_id}
ex : {{url}}/get_wishlist?user_id=2
리턴 정보 : (게임id, 게임 이름, 게임사진 작은버전)
게임은 br태그로 구분되어 있으며 게임 정보는 p태그로 구분되어서 제공됨.

위시리스트 추가
작동 방법 : add_wishlist?user_id={user_id}&game_id={game_id}
ex : {{url}}/add_wishlist?user_id=2&game_id=1
리턴 정보 : SUCCESS, FAIL
사용자의 위시리스트에 게임을 추가함

위시리스트 삭제
작동 방법 : del_wishlist?user_id={user_id}&game_id={game_id}
ex : {{url}}/del_wishlist?user_id=2&game_id=1
리턴 정보 : SUCCESS, FAIL
해당 게임을 사용자의 위시리스트에서 삭제함

차단목록
작동 방법 : get_block?user_id={user_id}
ex : {{url}}/get_block?user_id=2
리턴 정보 : (상대유저id, 상대방 닉네임, 상대 프사 작은버전)
사용자의 차단 목록 관리를 위한 정보임

상대방 차단하기
작동 방법 : add_block?user_id={user_id}&his_id={target_user_id}
ex : {{url}}/add_block?user_id=2&his_id=1
리턴 정보 : SUCCESS, FAIL
상대방을 차단 목록에 추가해서 서로 만나요에서 보이지 않고 상대방이 채팅을 보내도 보이지 않음

차단 해제
작동 방법 : del_block?user_id={user_id}&his_id={target_user_id}
ex : {{url}}/del_block?user_id=2&his_id=1
리턴 정보 : SUCCESS, FAIL
상대방을 차단 목록에서 삭제해서 다시 만나요에서 보거나 채팅을 칠 수 있음

저장한 카페 목록
작동 방법 : get_fav_cafe?user_id={user_id}
ex : {{url}}/get_fav_cafe?user_id=1
리턴 정보 : (카페id, 카페이름, 카페 사진 작은버전)

저장한 카페 추가
작동 방법 : add_fav_cafe?user_id={user_id}&cafe_id={cafe_id}
ex : {{url}}/add_fav_cafe?user_id=1&cafe_id=1
리턴 정보 : SUCCESS, FAIL

저장한 카페 삭제
작동 방법 : del_fav_cafe?user_id={user_id}&cafe_id={cafe_id}
ex : {{url}}/del_fav_cafe?user_id=1&cafe_id=1
리턴 정보 : SUCCESS, FAIL

유저 평가 추가, 수정
작동 방법 : add_usercomment?user_id={writter_id}&his_id={target_user_id}&score={score in range(0, 11)}&comment={comment}
ex : {{url}}/add_usercomment?user_id=1&his_id=2&score=10&comment=너무 좋아요
사용자가 상대방을 처음 평가할 경우 새로 생기고 이미 존재할 경우 수정해서 데이터베이스에 저장함.

유저 평가 불러오기
작동 방법 : get_usercomment?user_id={user_id}
ex : {{url}}/get_usercomment?user_id=2
리턴 정보 : 평점 평균, (작성자id, 작성자닉네임, 작성자프사, 평가점수, 평가내용)

회원가입
작동 방법 : join_user?platform={platform_number(email=0, kakao=1, naver=2)}&token={token(if platform is not 0: NOTNULL, else: NULL)} &email={email(if platform is not 0: NULLABLE)}&password={password(if platform is 0: NOTNULL, else: NULL)}&name={name(NULLABLE)} &nickname={nickname(NULLABLE)}&phone={phone_number(NULLABLE)}&age={age(NULLABLE)}
ex : {{url}}/join_user?platform=1&token=123456789&name=쿠민
platform은 필수로 전달해야 하고 이메일인 경우 email, password. 카카오나 네이버인 경우 token을 전달해야 한다.

로그인
작동 방법 : login?platform={platform_number(email=0, kakao=1, naver=2)}&token={token(if platform is not 0: NOTNULL, else: NULL)} &email={email(if platform is not 0: NULLABLE)}&password={password(if platform is 0: NOTNULL, else: NULL)}
ex : 이메일인경우 : {{url}}/login?platform=0&email=kookmin@gmail.com&password=wowwowowowwo
네이버나 카카오인 경우 : {{url}}/login?platform=2&token=123456789
